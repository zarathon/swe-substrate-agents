⚠️ O Grande Problema do Claude Code

Claude Code é uma ferramenta incrível, mas tem uma dependência crítica: CONTEXTO

🚨 Sem Contexto Estruturado, Claude Code Falha

Você já percebeu que às vezes o Claude Code gera código que não segue os padrões do seu projeto? Ou sugere arquiteturas completamente diferentes do que você usa?

Não é falha do Claude. É falta de contexto estruturado sobre o substrato de engenharia do seu projeto.

💡 Claude Code precisa entender as camadas fundamentais (substrato) da engenharia do seu software

❌ Sem Contexto Estruturado

  • Claude gera código genérico que não compila
  • Sugere bibliotecas que você não usa
  • Ignora convenções do time
  • Cria testes que não rodam
  • Propõe arquiteturas incompatíveis
  • Desperdiça tempo com refatorações

✅ Com Substrato Estruturado

  • Claude gera código idiomático que funciona
  • Usa as bibliotecas corretas do projeto
  • Segue todas as convenções estabelecidas
  • Cria testes que rodam imediatamente
  • Propõe soluções alinhadas com a arquitetura
  • Economiza horas de desenvolvimento

🧬 O Que é Substrato de Engenharia?

São as camadas fundamentais que definem como seu software é construído, testado, implantado e mantido. É o DNA do seu projeto.

🏗️ Arquitetura & Design

Como seu sistema é estruturado? REST ou GraphQL? Microserviços ou monolito? Que padrões você segue?

  • Patterns de arquitetura
  • Estilos de API
  • Estrutura de diretórios

🔧 Ferramental & Config

Como você configura o ambiente? Docker? Kubernetes? Que variáveis de ambiente são críticas?

  • Containerização
  • IaC (Infrastructure as Code)
  • Scripts de setup

📝 Convenções de Código

Como você escreve código? Que idiomática segue? Como gerencia concorrência e memória?

  • Style guides
  • Padrões de naming
  • Error handling

✅ Estratégia de Testes

Como você testa? Que frameworks usa? Qual coverage esperado? Como rodam os testes?

  • Unitários, Integração, E2E
  • Mocks e fixtures
  • CI/CD pipelines

💾 Dados & Persistência

Como você persiste dados? PostgreSQL? MongoDB? Como faz migrations? Cache strategies?

  • Databases e schemas
  • ORMs e query builders
  • Caching layers

🔐 Segurança

Como você autentica? JWT? OAuth? Como gerencia secrets? Certificados SSL?

  • Auth strategies
  • Secrets management
  • Security practices

🚀 Como Estruturar seu Substrato (3 minutos)

1

Clone e copie para seu projeto

git clone https://github.com/zarathon/swe-substrate-agents.git cd swe-substrate-agents cp -r .claude /seu-projeto/
2

Execute o orquestrador no Claude Code

/swe-substrate-init

Ele lista todos os agentes disponíveis e explica o sistema

3

Execute os agentes relevantes

/swe-substrate-languages /swe-substrate-testing-strategy /swe-substrate-data /swe-substrate-security

Execute apenas os agentes relevantes para seu projeto. Cada um fará perguntas abertas.

4

Responda as perguntas com contexto

Os agentes combinam análise automática do código com suas respostas para gerar documentação estruturada em .claude/swe-substrate/

✨ Por Que Modularizar?

📦 Escalabilidade

Documentação cresce sem limites. Cada módulo é independente e pode ser atualizado separadamente.

🔧 Manutenção

Fácil de navegar, atualizar e revisar. Sem arquivo CLAUDE.md monolítico gigante.

🤝 Colaboração

Menos conflitos em PRs. Times podem trabalhar em módulos diferentes simultaneamente.

⚡ Performance

Claude Code carrega apenas o contexto relevante para cada tarefa, otimizando velocidade.
